SUB subName(param1 AS type, param2 AS type, ...)
     {code}
END SUB


Description:

Subs are blocks of code that are seperate from the rest of the program code, and can be called from anywhere within a program using the CALL command. They are much like Functions, except they do not return anything. subName specifies the name by which the sub will called. A sub can have in any number of parameters, each of which is specified after subName much like a varible declaration in the VAR command. Subs can be placed anywhere within the program code, since the compiler clips all function and sub definitions out before running the program. The only place a sub definition cannot be is within another sub or function definition. All variables and arrays declared within a sub can only be used by the {code} within the sub. Things such as window names and file handles are global, and can be seen anywhere in the program code. Globally declared variables and arrays are also recognized within all subs and functions, but are recognized only if there is not a local variable or array of the same name. For example, if a varaible called num was declared globally (not in a sub or function), as well as in a the sub GetVal, the local version of num (inside of GetVal) will be recognized instead of the global version.
